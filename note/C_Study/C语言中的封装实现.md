# C 语言中的封装实现总结

封装（Encapsulation）是面向对象编程的核心概念之一。虽然 C 语言本身不是面向对象的语言，但可以通过结构体、函数指针和编程约定来实现类似的封装效果。基于本项目中的 `led` 模块代码，我们可以总结出以下几种 C 语言实现封装的常用手段。

## 1. 结构体：数据与行为的集合
在 C 语言中，封装的第一步是将相关的数据（属性）组合在一起。

```c
typedef struct led {
    BaseOps ops;  // 封装行为（函数指针）
    int pin;      // 封装数据（硬件引脚）
} LED;
```
通过这种方式，`LED` 结构体成为了一个“类”的载体，它不仅包含了状态信息（`pin`），还包含了操作这些状态的“方法”集合（`ops`）。

## 2. 函数指针：抽象接口与多态
项目使用了 `BaseOps` 结构体来定义一组标准的操作接口。这种方式模拟了面向对象中的“接口”或“虚函数表（VTable）”。

```c
typedef struct base_ops {
    void (*init)(void *self);
    void (*on)(void *self);
    void (*off)(void *self);
    void (*toggle)(void *self);
} BaseOps;
```
*   **接口统一**：无论底层是 LED A、B 还是 C，外部调用者都只需要关注 `init`、`on` 等函数名。
*   **隐藏实现**：具体的实现函数（如 `led_onA`）可以定义在 `.c` 文件中，通过函数指针暴露给外部，从而实现行为的封装。

## 3. `self` 指针：模拟实例上下文
在 C 语言的“方法”实现中，无法像 C++ 那样自动获得 `this` 指针。本项目通过显式传递 `void *self` 来模拟这一机制。

```c
void led_onA(void *self) {
    LED *led = (LED *)self; // 类型转换，重新获得上下文
    printf("LED on pin %d is now on\n", led->pin);
}
```
通过这种显式的上下文传递，函数能够准确地操作属于特定“实例”的数据，实现了数据与函数的逻辑绑定。

## 4. 构造函数模式：对象初始化封装
通过提供专门的创建函数（如 `led_create`），我们将内存分配、状态初始化和接口绑定封装在了一个单一的入口中。

```c
LED* led_create(int pin, BaseOps ops) {
    LED *led = (LED*)malloc(sizeof(LED));
    if (led != NULL) {
        led->pin = pin;
        led->ops = ops; // 绑定特定的实现
    }
    return led;
}
```
外部使用者只需调用 `led_create` 即可获得一个完整的、可工作的对象，而无需关心内部复杂的配置过程。

## 5. 模块化封装（H/C 文件分离）
这是 C 语言最基础也是最有效的封装手段：
*   **`.h` 文件**：声明公共接口和结构体类型，作为对外公开的“说明书”。
*   **`.c` 文件**：实现具体的逻辑细节。
*   **补充建议**：如果想要更彻底的封装（信息隐藏），可以将 `struct LED` 的具体定义放在 `.c` 文件中，而在 `.h` 文件中仅保留 `typedef struct led LED;` 的不透明指针声明。

## 总结
本项目展示了 C 语言如何通过**结构体封装数据**、**函数指针封装行为**、**构造函数封装逻辑**来实现一个灵活且易于扩展的硬件抽象层（HAL）。这种设计模式不仅增强了代码的可读性，还为多态（如支持不同型号的 LED）打下了基础。
