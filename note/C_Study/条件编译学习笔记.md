# C 语言条件编译 (Conditional Compilation)

条件编译是指预处理器在编译代码之前，根据宏定义或条件表达式的结果，决定哪些代码段参与编译，哪些代码段被忽略。

## 核心预处理指令

1.  `#ifdef MACRO`: 如果定义了宏 `MACRO`，则编译后续代码。
2.  `#ifndef MACRO`: 如果没有定义宏 `MACRO`，则编译后续代码（常用作 Header Guard）。
3.  `#if expression`: 如果表达式为真，则编译后续代码。
4.  `#elif expression`: 相当于 `else if`。
5.  `#else`: 相当于 `else`。
6.  `#endif`: 结束条件编译块。
7.  `defined(MACRO)`: 在 `#if` 中使用，检查宏是否定义。

## 常见用途

-   **调试开关**: 在开发阶段开启 `DEBUG` 日志，发布时关闭。
-   **跨平台兼容**: 根据操作系统宏（如 `_WIN32`, `__linux__`）编写不同的底层代码。
-   **功能剪裁**: 根据编译选项启用或禁用特定模块。
-   **头文件防重复包含 (Header Guard)**:
    ```c
    #ifndef MY_HEADER_H
    #define MY_HEADER_H
    // 头文件内容
    #endif
    ```

    在 C++ 中，除了传统的 `#ifndef` 方式，还可以使用 `#pragma once`，这是更简洁的现代方式（非标准但被大多数编译器支持）：
    ```cpp
    #pragma once
    // 头文件内容
    ```

## 编译示例

可以使用 `-D` 选项在编译时注入宏：
```bash
gcc -DDEBUG=1 example.c -o example
```
