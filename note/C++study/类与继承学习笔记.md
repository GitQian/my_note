# C++ 类与继承学习笔记 —— 核心机制与实战

本笔记基于 `class_test` 项目，深入剖析 C++ 面向对象编程（OOP）的核心概念：继承方式、多态实现、常量属性以及内存管理。

---

## 1. 继承方式的选择 (Inheritance Modes)
C++ 提供了三种继承方式，决定了基类成员在派生类中的最高访问权限。

| 继承方式 | 基类 Public | 基类 Protected | 基类 Private | 隐式上转型 (Upcasting) |
| :--- | :--- | :--- | :--- | :--- |
| **public** | 保持 public | 保持 protected | 不可访问 | **允许** (接口继承) |
| **protected** | 变为 protected | 变为 protected | 不可访问 | 仅子类/友元允许 |
| **private** | 变为 **private** | 变为 **private** | 不可访问 | **禁止** (实现继承) |

**代码实战：**
- `SubClassA` 是 **公有继承**：支持多态，`Base *p = new SubClassA()` 是合法的。
- `SubClassB` 是 **私有继承**：仅内部使用基类逻辑，外部无法将 `SubClassB` 当作 `Base` 使用。

## 2. 虚函数与多态 (Virtual Functions & Polymorphism)
### 2.1 抽象类与纯虚函数
如果类中包含至少一个**纯虚函数**（如 `virtual int pub_fun() = 0;`），该类即为**抽象类**，无法实例化。

### 2.2 虚析构函数的必要性
**重要知识点**：基类的析构函数必须声明为 `virtual`。
- **原因**：当使用基类指针删除派生类对象时（如 `Base *p = new SubClassA(); delete p;`），如果析构函数不是虚函数，编译器只会调用 `Base` 的析构，导致派生类的内存泄漏。

### 2.3 `override` 关键字 (C++11)
在子类重写虚函数时加上 `override`，可以让编译器协助检查函数签名是否与基类完全一致，避免因拼写错误导致的“隐形 Bug”。

## 3. 常函数与 `mutable` 关键字
### 3.1 常函数 (const Member Functions)
- **定义**：`int func() const;`
- **限制**：常函数承诺不修改对象的任何非静态成员变量。
- **调用规则**：常对象（`const Obj`）**只能**调用常函数。

### 3.2 `mutable` 突破限制
有时候我们需要在常函数中修改某些与对象状态无关的变量（如缓存、互斥锁、计数器）。
- **作用**：被 `mutable` 修饰的变量即使在 `const` 函数中也可以被修改。
```cpp
mutable int mut_a; // 在 SubClassB::pub_fun_const() 中被修改
```

## 4. 对象创建的多种方式
代码中演示了 7 种对象处理方式，推荐优先级如下：
1. **智能指针** (`std::unique_ptr`)：自动管理内存，防止泄漏。
2. **栈分配**：性能最高，作用域结束自动销毁。
3. **引用/指针多态**：实现接口统一调用的核心。

## 5. 补充知识点 (重要)

### 5.1 虚函数表原理 (vtable)
- 只要类中有虚函数，编译器就会为该类创建一个 **vtable**（存储虚函数地址）。
- 每个对象实例中包含一个隐藏指针 **vptr**，指向所属类的 vtable。
- **性能开销**：多态调用会有一次指针跳转的开销，通常可忽略。

### 5.2 拷贝构造与深浅拷贝
- **浅拷贝**：编译器默认提供，仅逐成员赋值。如果成员有指针指向堆内存，会导致多个对象释放同一块内存。
- **深拷贝**：当类中有堆内存分配时，**必须**自定义拷贝构造函数和赋值运算符。

### 5.3 对象切片 (Object Slicing)
当把子类对象**按值**赋值给基类对象时，会发生切片：
```cpp
Base b = SubClassA(); // 子类特有的部分被“切掉”了
```
**规避**：始终通过**指针**或**引用**来传递对象以维持多态性。

---

## 6. 常见错误总结 (基于代码注释)
1. **私有访问限制**：子类可以继承但不能直接访问父类的 `private` 成员。
2. **作用域访问**：`Base::pub_fun_test()` 可以强制调用父类实现，但在私有继承下，外部无法通过 `b_obj.Base::...` 访问。
3. **拷贝构造返回**：拷贝构造函数没有返回值，它的职责是**初始化**。
