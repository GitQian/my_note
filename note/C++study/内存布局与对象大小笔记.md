# C++ 对象内存布局与大小 (Memory Layout & Object Size)

## 1. 核心结论：什么占用空间？
一个 C++ 对象的大小由以下部分组成：
1.  **非静态数据成员**：类的普通变量。
2.  **虚函数指针 (vptr)**：如果类包含 `virtual` 函数，每个对象会多一个指针大小的空间。
3.  **为了对齐而填充的字节 (Padding)**：编译器为了 CPU 读取效率，会在成员间填充空白。
4.  **父类的数据成员**：继承自父类的部分。

**不占用对象空间的部分：**
*   **成员函数**：代码存储在程序的代码段，所有对象共享。
*   **静态成员 (static)**：存储在全局/静态区，所有对象共享一份。

---

## 2. 空类为什么是 1 字节？
```cpp
class Empty {}; // sizeof(Empty) == 1
```
如果空类大小为 0，那么 `Empty a, b;` 两个对象的地址就会相同。为了确保**“每个对象在内存中都有唯一的地址”**，C++ 规定空类至少占用 1 字节。

---

## 3. 内存对齐 (Data Alignment & Padding)
CPU 通常以字（word，如 64 位系统是 8 字节）为单位读取内存。如果数据跨越了字边界，效率会降低。
编译器遵循以下规则：
1.  **成员对齐**：每个成员相对于对象起始地址的偏移量，必须是该成员大小（或 `alignas` 指定大小）的倍数。
2.  **整体对齐**：对象的总大小必须是类中最大基本类型成员大小的倍数。

**示例对比：**
```cpp
class A { char a; int b; char c; }; // 1 + (3填充) + 4 + 1 + (3填充) = 12 字节
class B { int b; char a; char c; }; // 4 + 1 + 1 + (2填充) = 8 字节
```

---

## 4. 虚函数的影响 (The vptr)
当类中定义了第一个虚函数时，编译器会为该类创建一个**虚函数表 (vtable)**，并在每个对象中插入一个**虚函数指针 (vptr)**，指向该表。
*   在 64 位系统上，`sizeof(vptr)` 是 8 字节。
*   无论有多少个虚函数，对象中通常只有一个 `vptr`。

---

## 5. 继承中的大小
*   **普通继承**：子类的大小 = 父类成员大小 + 子类成员大小 + 对齐填充。
*   **虚继承 (Virtual Inheritance)**：用于解决菱形继承问题。这会引入额外的**虚基类指针 (vbptr)**，显著增加对象大小。

---

## 6. 工具推荐
如果你想深入查看类在内存中的布局，可以使用以下编译器参数：
*   **Clang**: `-Xclang -fdump-record-layouts`
*   **MSVC**: `/d1reportSingleClassLayoutXXX`
*   **GCC**: `-fdump-lang-class` (输出到文件)

---

## 7. 总结：如何减小类的大小？
1.  **按大小降序排列成员**：先放 8 字节的，再放 4 字节的，最后放 1 字节的。
2.  **避免不必要的虚函数**：如果不需要多态，就不要用 `virtual`。
3.  **使用位域 (Bit-fields)**：如果某些变量只需要几个 bit（如 bool 开关）。
