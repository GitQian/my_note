# C++ 异步与并发编程学习笔记 —— 生产者-消费者模型

本笔记基于 `Day3` 的生产者-消费者代码，总结了 C++11/17 中并发编程的核心组件。

---

## 1. 条件变量 (std::condition_variable)
条件变量用于线程间的同步，允许一个线程等待另一个线程的信号。

### 核心机制：
- **`wait(lock, predicate)`**：
  1. 释放 `unique_lock`。
  2. 阻塞当前线程，直到被通知（notify）。
  3. 被唤醒后尝试重新获取锁，并检查 `predicate`（谓词）是否为真。如果为假，继续等待。
- **`notify_one()` / `notify_all()`**：唤醒一个或所有正在等待的线程。

**代码片段：**
```cpp
// 典型的等待模式
cv_.wait(lock, [this]() { return !queue_.empty() || done_; });
```

---

## 2. 互斥锁 (Mutex & Locks)
- **`std::mutex`**：最基础的互斥量。
- **`std::lock_guard`**：RAII 风格的锁，构造加锁，析构解锁。适用于简单的代码块保护。
- **`std::unique_lock`**：比 `lock_guard` 更灵活，支持手动解锁、延迟加锁，且是 **`condition_variable::wait` 必须使用的锁类型**。

---

## 3. 异步任务 (std::async & std::future)
相比于传统的 `std::thread`，`std::async` 提供了更高级的抽象。

- **`std::async`**：启动一个异步任务。建议显式指定 `std::launch::async` 以确保在新线程中运行。
- **`std::future`**：代表一个未来的值。通过 `get()` 方法获取结果，**如果结果还未产出，`get()` 会阻塞当前线程**。

**重要知识点：**
`std::async` 返回的 `std::future` 对象如果没被接收（即作为临时对象），其析构函数会阻塞直到异步任务完成。

---

## 4. 线程安全队列的实现逻辑
一个工业级的线程安全队列需要考虑：
1. **原子性**：入队和出队操作必须加锁。
2. **通知机制**：入队后通知消费者，生产结束后通知所有等待者。
3. **防止虚假唤醒**：通过 `wait` 的第二个参数（谓词）来确保条件确实满足。

---

## 5. 补充知识点 (重要)

### 5.1 数据竞争 (Data Race)
当两个或多个线程同时访问同一个内存位置，且至少有一个是写操作时，会发生数据竞争，导致未定义行为。

### 5.2 死锁 (Deadlock)
当两个线程互相等待对方释放锁时会发生死锁。
- **预防**：始终以相同的顺序获取多个锁，或使用 `std::lock(m1, m2)` 一次性获取多个锁。

### 5.3 线程池的概念
代码中每次消费任务都开启一个新的 `std::async`。在实际生产中，频繁创建/销毁线程开销巨大，通常使用**线程池**（Thread Pool）来复用固定数量的线程。
