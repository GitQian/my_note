# C++ STL 容器深度解析笔记

## 1. 为什么有些容器有“特有函数”？
核心逻辑：**效率。**
虽然 `<algorithm>` 中的通用算法能解决大部分问题，但它只通过**迭代器**进行操作。而容器的**成员函数**知道容器的内部实现细节（如链表结构、树结构、哈希表结构），因此能提供更优的性能。

---

## 2. 关键对比：通用算法 vs 成员函数

### 2.1 排序 (Sort)
*   **`std::sort`**：要求随机访问迭代器 (`RandomAccessIterator`)。
    *   **适用**：`vector`, `deque`, `array`。
    *   **不适用**：`list`, `forward_list`, `set/map`。
*   **`list::sort()`**：链表内部通过改变指针指向来排序，比拷贝数据快得多。

### 2.2 查找 (Find)
*   **`std::find`**：线性查找 (O(n))。
*   **`set::find` / `map::find`**：二分查找 (O(log n))。
    *   **原因**：通用 `std::find` 只会一个一个往后走。成员 `find` 知道树的结构，可以左右跳。

### 2.3 计数 (Count)
*   **`std::count`**：遍历整个容器 (O(n))。
*   **`set::count`**：利用有序性快速定位 (O(log n))。

---

## 3. 容器三剑客
| 容器类型 | 代表 | 核心特点 | 性能 |
| :--- | :--- | :--- | :--- |
| **序列容器** | `vector` | 连续内存，像数组 | 随机访问 O(1)，末尾增删快 |
| **双向链表** | `list` | 离散内存，指针连接 | 随机访问 O(n)，任意位置增删 O(1) |
| **关联容器** | `set`/`map` | 有序，红黑树实现 | 查找/插入/删除均 O(log n) |
| **无序容器** | `unordered_map` | 哈希表实现 | 查找平均 O(1)，最差 O(n) |

---

## 4. 开发陷阱：`std::map` 的 `[]`
*   **特性**：`map[key]` 如果找不到 key，会调用默认构造函数**自动插入**一个新键值对。
*   **后果**：这可能导致你原本只想“查询”是否存在，却意外改变了容器内容。
*   **方案**：只读场景请使用 `find()`。

---

## 5. 什么时候该用哪个？
1.  **默认选择**：`std::vector`。它的内存连续性对 CPU 缓存最友好。
2.  **需要大量中间增删**：`std::list`。
3.  **需要自动去重或排序**：`std::set`。
4.  **需要极速查找键值对**：`std::unordered_map`。

---

## 6. 进阶：如何选择成员函数还是算法？
**记住一句话：如果有成员函数版本，优先用成员函数。**
因为它不仅更快，通常语义也更清晰，还能避免因迭代器类型不匹配导致的编译错误。
