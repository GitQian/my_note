# C++ 仿函数 (Functors) 与 std::function 学习笔记

## 1. 仿函数 (Functors)
仿函数（Functor）是一个重载了 `operator()` 的类或结构体。它的行为类似于函数，但它是一个对象，因此可以拥有状态。

### 优点
- **状态保存**：仿函数可以拥有成员变量来保存状态（如计数器、配置参数等）。
- **性能**：由于仿函数是对象，编译器更容易对其进行内联优化（Inline Optimization），通常比函数指针快。
- **类型安全**：每个仿函数都有自己的类型，可以作为模板参数传递。

### 示例
```cpp
class Add {
public:
    int operator()(int a, int b) const { return a + b; }
};
```

## 2. std::function
`std::function` 是 C++11 引入的一个通用的、多态的函数包装器。

### 特点
- **统一性**：可以存储、复制和调用任何可调用对象（Callable Targets），包括普通函数、Lambda 表达式、仿函数、以及通过 `std::bind` 绑定的函数。
- **开销**：相比于直接调用函数或使用模板，`std::function` 有一定的运行时开销（通常涉及虚函数调用或动态内存分配）。

## 3. Lambda 表达式
Lambda 表达式是匿名函数的简写形式，其实质是编译器自动生成的仿函数。

### 捕获列表
- `[]`：不捕获任何变量。
- `[=]`：按值捕获作用域内的所有变量。
- `[&]`：按引用捕获作用域内的所有变量。
- `[x]`：按值捕获变量 x。

## 4. 扩展补充
### std::function vs 模板 (Templates)
- **模板**：在编译期确定类型，性能最高，但会增加编译时间和二进制大小。
- **std::function**：在运行期提供多态性，更加灵活，但有轻微的性能损耗。

### std::bind
`std::bind` 可以用来绑定函数的参数，将一个多元函数转换为低元函数，常用于事件处理和回调机制。

## 5. 类成员函数的绑定
存储类成员函数时，`std::function` 需要知道函数所属的对象实例（即 `this` 指针）。

### 方法 A：使用 std::bind (传统)
```cpp
auto func = std::bind(&MyClass::memberFunc, &obj, std::placeholders::_1);
```

### 方法 B：使用 Lambda 表达式 (推荐)
```cpp
auto func = [&obj](int arg) { return obj.memberFunc(arg); };
```

### 方法 C：显式传递对象指针 (延迟绑定)
如果不绑定具体实例，可以将对象指针作为 `std::function` 的第一个参数：
```cpp
std::function<void(MyClass*, int)> func = &MyClass::memberFunc;
func(&obj, 10); // 调用时传入对象
```

## 6. std::function vs 原始函数指针
为什么在现代 C++ 中我们更倾向于使用 `std::function` 而不是传统的函数指针（如 `void (*ptr)(int)`）？

### 1. 灵活性与通用性
- **函数指针**：只能指向普通的全局函数或静态成员函数。它无法存储 **Lambda 表达式（带捕获）**、**仿函数** 或 **类成员函数**。
- **std::function**：是一个“万能包装器”。只要签名匹配，它可以存储任何可调用对象（Callable Objects）。

### 2. 状态保存
- **函数指针**：本身不具备状态。如果你需要函数带上一些上下文数据，通常需要额外的 `void*` 参数。
- **std::function**：可以通过 Lambda 捕获或仿函数成员变量轻松携带状态，实现“闭包”的效果。

### 3. 类成员函数的处理
- **函数指针**：指向成员函数的指针语法极其复杂（如 `int (MyClass::*ptr)(int)`），且调用时必须显式绑定对象。
- **std::function**：可以将成员函数与对象实例绑定后，包装成一个普通的 `std::function<int(int)>`，调用者甚至不需要知道后端是一个成员函数。

### 4. 性能权衡
- **函数指针**：性能略高，因为它只是一个内存地址的跳转，没有类型擦除（Type Erasure）和动态分配的开销。
- **std::function**：由于使用了类型擦除技术，通常会有微小的运行时开销（类似虚函数调用）。但在现代硬件和绝大多数业务逻辑中，这种开销几乎可以忽略不计。

### 总结：什么时候用谁？
- **优先使用 `std::function`**：当你需要回调、事件驱动或高度解耦的设计时。
- **使用函数指针**：当你编写极高性能要求的底层库，或者需要与 C 语言接口对接时。
