# C++ 移动语义 (Move Semantics) 学习笔记

## 1. 核心背景：为什么要引入移动语义？
在 C++11 之前，如果你想把一个包含大量内存资源的对象传递给另一个对象，通常只能通过**拷贝**（Copy）。
*   **深拷贝的代价**：如果对象很大（比如一个包含 100 万个元素的 `vector`），深拷贝会分配新内存并逐一复制元素，极其耗时。
*   **临时对象的浪费**：很多时候，原对象在拷贝后就不再使用了（比如函数返回的临时对象），这种“先拷贝、再销毁原对象”的操作是巨大的资源浪费。

**移动语义的核心思想**：**“偷”取资源**。不再复制内存，而是直接把原对象的指针接管过来，并将原对象置为空。

---

## 2. 左值 (Lvalue) vs 右值 (Rvalue)
理解移动语义的前提是分清这两种值：
*   **左值 (Lvalue)**：有名字、有固定内存地址的对象。可以取地址 `&`。
    *   例如：变量名 `int a = 10;` 中的 `a`。
*   **右值 (Rvalue)**：匿名的、临时的、没有固定内存地址的对象。无法取地址。
    *   例如：字面量 `10`，表达式的结果 `a + b`，或者函数的临时返回值。

### 右值引用 (`T&&`)
C++11 引入了 `&&` 语法，专门用来绑定到右值上：
```cpp
int&& ref = 10; // 正确：右值引用绑定到临时右值
// int&& ref_a = a; // 错误：右值引用不能直接绑定到左值变量
```

---

## 3. `std::move` 的真相
`std::move` 并不移动任何东西！
它的唯一作用是：**强制类型转换**。它将一个**左值**（有名字的对象）转换为**右值引用**。
*   当你调用 `std::move(obj)` 时，你是在告诉编译器：“我已经不再需要这个 `obj` 了，你可以随便偷它的资源。”

---

## 4. 移动构造与移动赋值
为了支持移动语义，你的类需要实现这两个特殊的成员函数：

### 移动构造函数
```cpp
Resource(Resource&& other) noexcept {
    this->data = other.data;  // 接管指针
    other.data = nullptr;     // 将源对象置空（防止析构时释放资源）
}
```

### 移动赋值运算符
```cpp
Resource& operator=(Resource&& other) noexcept {
    if (this != &other) {
        delete this->data;    // 释放自己的旧资源
        this->data = other.data; // 偷取新资源
        other.data = nullptr;    // 将源对象置空
    }
    return *this;
}
```

**关键点：**
1.  **参数必须是 `T&&`**：接收右值。
2.  **`noexcept`**：强烈建议添加。如果移动构造函数不保证不抛出异常，`std::vector` 在扩容时可能为了保证强异常安全性而回退到使用拷贝构造。
3.  **源对象状态**：移动后，源对象必须处于一个**“有效但未定义”**的状态。最常见的做法是将指针置为 `nullptr`。

---

## 5. 性能提升
通过 `move_semantics_test.cpp` 的对比可以看到：
*   **拷贝构造**：耗时取决于数据量（O(n)）。
*   **移动构造**：耗时几乎为常数级（O(1)），因为它只涉及几个指针的赋值。

---

## 6. 什么时候使用？
1.  **容器操作**：`vector.push_back(std::move(obj))` 可以避免拷贝。
2.  **函数返回大型对象**：现代编译器有 RVO（返回值优化），但移动语义提供了保底的性能支持。
3.  **管理独占资源**：如 `std::unique_ptr`，它只能移动，不能拷贝。
