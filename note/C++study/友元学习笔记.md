# C++ 友元 (Friend) 学习笔记

友元机制允许一个函数或类访问另一个类的 **private** 和 **protected** 成员。它是对封装性的一种“特权”开放，通常用于某些需要高度紧密协作的场景。

---

## 1. 友元的三种实现方式

### 1.1 全局函数做友元
在类内部用 `friend` 关键字声明全局函数，该函数即可访问类的私有成员。
```cpp
class A {
    friend void friend_func(A &a); // 声明全局函数为友元
private:
    int a_priv;
};

void friend_func(A &a) {
    cout << a.a_priv; // 合法访问
}
```

### 1.2 类做友元
当类 B 被声明为类 A 的友元时，类 B 的**所有成员函数**都可以访问类 A 的私有成员。
```cpp
class A {
    friend class B; // 声明类 B 为友元
private:
    int priv_a;
};

class B {
public:
    void getPriv(A &a) { cout << a.priv_a; } // 合法
};
```

### 1.3 成员函数做友元
只允许特定的某个类的成员函数访问私有成员，而不是整个类。
- **注意**：这种方式对代码的声明顺序有严格要求。通常需要先声明被访问的类，再完整定义友元函数所在的类。
```cpp
class A; // 前置声明
class C {
public:
    void getPriv(A &a); 
};

class A {
    friend void C::getPriv(A &a); // 仅指定 C 的这个函数为友元
private:
    int priv_a;
};
```

## 2. 友元的核心特性 (重要)

友元关系具有以下三个核心特性，面试中经常涉及：

1.  **单向性**：
    - 如果 A 是 B 的友元，并不意味着 B 是 A 的友元。特权是单向授予的。
2.  **不可传递性**：
    - 如果 A 是 B 的友元，B 是 C 的友元，并不意味着 A 是 C 的友元。“你朋友的朋友，不一定是你的朋友”。
3.  **不继承性**：
    - 基类的友元不会自动成为派生类的友元。同样，派生类的友元也不能访问基类的私有成员。

---

## 3. 典型应用场景：运算符重载

最常见的友元使用场景是重载 **左移运算符 `<<`**（用于 `cout` 输出对象）。
- 因为 `cout` 是 `ostream` 类型的对象，且必须作为左操作数。如果不用友元，就无法在不修改标准库的情况下访问自定义类的私有数据。

```cpp
class Point {
    friend ostream& operator<<(ostream& out, const Point& p);
private:
    int x, y;
};

ostream& operator<<(ostream& out, const Point& p) {
    out << "(" << p.x << ", " << p.y << ")";
    return out;
}
```

---

## 4. 优缺点分析

- **优点**：
  - 提高了程序的灵活性，方便类与类、类与函数之间的数据交换。
  - 减少了不必要的 Getter/Setter 函数调用，简化了代码。
- **缺点**：
  - **破坏了封装性**：友元可以绕过访问控制，增加了代码维护的风险。
  - 增加了类之间的**耦合度**：友元关系的改变可能导致多处代码需要重构。

**开发建议**：不到万不得已，尽量少用友元。如果必须使用，优先考虑“成员函数做友元”，以缩小影响范围。
