# C++ 线程基础 (std::thread) 学习笔记

## 1. std::thread 基本用法
`std::thread` 是 C++11 提供的多线程支持库，位于 `<thread>` 头文件中。

### 线程生命周期管理
- **join()**：主线程等待子线程执行完毕后再继续。
- **detach()**：子线程在后台独立运行，主线程无需等待其结束。

### 注意事项
1. **joinable()**：在调用 `join()` 或 `detach()` 之前，最好检查线程是否可以 join。
2. **RAII**：如果不调用 `join()` 或 `detach()`，`std::thread` 对象在析构时会调用 `std::terminate()`，导致程序崩溃。

## 2. 线程参数传递
`std::thread` 的构造函数可以接收任意数量的参数。

### 参数拷贝与引用
- `std::thread` 的构造函数默认通过**按值拷贝**的方式传递参数。
- 如果需要通过**按引用传递**，必须显式地使用 `std::ref(arg)` 包装参数。

### 示例
```cpp
void increment(int& n) { n++; }
int x = 0;
std::thread t(increment, std::ref(x)); // 正确方式
```

## 3. 常见坑点与扩展
### detach() 的危险性
- 线程脱离主线程后，如果它访问了主线程中已销毁的局部变量（例如栈上的变量引用），会导致未定义行为（Undefined Behavior）。
- 建议尽量使用 `join()`，除非你确定后台线程在整个生命周期内不依赖主线程的局部资源。

### std::this_thread
- `std::this_thread::get_id()`：获取当前线程的 ID。
- `std::this_thread::sleep_for(duration)`：让当前线程睡眠指定时间。
- `std::this_thread::yield()`：建议操作系统重新调度，允许其他线程运行。

### hardware_concurrency()
- `std::thread::hardware_concurrency()`：返回系统支持的最大并发线程数（通常等于 CPU 的逻辑核心数）。这在实现线程池或动态调整并发量时非常有用。
