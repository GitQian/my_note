# C++ 模板 (Template) 学习笔记

模板是 C++ 泛型编程（Generic Programming）的核心，允许程序员编写与类型无关的代码。

---

## 1. 函数模板 (Function Templates)

### 1.1 基本语法
```cpp
template <typename T>
void mySwap(T &a, T &b) {
    T tmp = a;
    a = b;
    b = tmp;
}
```

### 1.2 调用方式
1.  **自动类型推导**：`mySwap(a, b);` 编译器根据实参自动推断 `T`。
2.  **显示指定类型**：`mySwap<int>(a, b);` 直接告诉编译器使用哪种类型。

### 1.3 调用优先级规则
- **普通函数优先**：如果模板函数和普通函数同时匹配，优先调用普通函数。
- **强制模板调用**：通过空模板参数列表 `mySwap<>(a, b);` 强制调用模板版本。
- **更好的匹配优先**：如果模板函数能产生更好的匹配（例如普通函数需要隐式转换而模板不需要），则优先调用模板。

---

## 2. 类模板 (Class Templates)

### 2.1 与函数模板的区别
- **无法自动推导**：类模板在使用时**必须**显示指定类型（C++17 之前）。
  - `Person<string, int> p("Alice", 20);`
- **可以有默认参数**：
  - `template <class Name, class Age = int>`

### 2.2 类模板成员函数的创建时机
类模板中的成员函数并不是一开始就生成的，而是在**调用时才生成**。
- **后果**：如果模板类内部调用了某个类型不具备的方法（如 `anim.miao()`），只要不调用对应的成员函数，编译就不会报错。一旦调用且类型不匹配，则会发生编译错误。

---

## 3. 类模板成员函数的类外实现

当在类外定义模板类的成员函数时，必须带上完整的模板参数列表：
```cpp
// 声明
template <class T>
class Anim {
    void jiao1();
};

// 类外实现
template <class T>
void Anim<T>::jiao1() {
    // ...
}
```

## 4. 补充重要知识点

### 4.1 模板的“二阶段编译”
1.  **模板定义阶段**：检查基本语法（如括号匹配、分号等）。
2.  **模板实例化阶段**：当确定了具体类型 `T` 后，再次检查类型相关的语法（如 `T` 是否支持 `+` 运算或某个成员函数）。

### 4.2 为什么模板通常定义在 `.h` 文件中？
由于编译器在实例化模板时需要看到完整的源代码，通常不建议将模板的声明和实现分离到 `.h` 和 `.cpp` 中（否则会导致链接错误）。
- **解决方案**：将实现也写在头文件中，或者使用 `.hpp` 后缀。

### 4.3 模板特化 (Specialization)
当某个特定类型不适合通用模板逻辑时，可以定义特化版本：
```cpp
// 通用模板
template<class T> bool compare(T a, T b) { return a == b; }

// 为 char* 提供特化版本（处理字符串比较）
template<> bool compare<char*>(char* a, char* b) {
    return strcmp(a, b) == 0;
}
```

### 4.4 模板参数中的非类型参数
模板不仅可以接收类型，还可以接收常量：
```cpp
template<class T, int size>
class MyArray {
    T arr[size]; // 静态数组
};
MyArray<int, 10> array;
```

---

## 5. 总结建议
- **代码重用**：对于逻辑相同但类型不同的函数/类，优先使用模板。
- **性能**：模板在编译期展开，**没有运行期开销**（对比多态），但可能会导致生成的二进制文件体积变大（模板膨胀）。
- **可读性**：复杂的模板报错信息通常很难读懂，建议分阶段测试。
