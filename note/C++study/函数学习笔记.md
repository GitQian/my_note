# C++ 函数高级特性学习笔记

本文档总结了 C++ 函数的核心特性，包括默认参数、占位参数、函数重载及其底层逻辑。

---

## 1. 默认参数 (Default Parameters)
C++ 允许在声明或定义函数时为参数提供默认值。

### 核心规则：
1. **尾部对齐原则**：如果某个位置的参数有默认值，那么从该位置往后，**从左向右**的所有参数都必须有默认值。
   ```cpp
   // 正确
   void func(int a, int b = 10, int c = 20); 
   // 错误：b 有默认值，c 也必须有
   // void func(int a, int b = 10, int c); 
   ```

2. **声明与实现不共存**：如果函数声明时已经写了默认参数，函数实现（定义）时就**不能**再写默认参数，否则会报“重定义默认参数”的错误。
   ```cpp
   // 声明
   int func_test2(int a, int b = 10);
   // 实现（不能再写 = 10）
   int func_test2(int a, int b) { ... }
   ```

---

## 2. 占位参数 (Placeholder Parameters)
C++ 支持在参数列表中只写类型而不写变量名。

- **语法**：`void func(int a, int) { ... }`
- **用途**：
  1. 兼容旧代码。
  2. 预留扩展接口。
  3. **运算符重载**（区分前置 ++ 和后置 ++）。
- **注意**：在函数定义处，通常需要给占位参数命名才能在函数体内使用它。

---

## 3. 函数重载 (Function Overload)
函数名相同，但参数列表不同（类型、个数、顺序）。

### 3.1 引用作为重载条件
这是重载中较为复杂的部分，编译器会根据传入的是左值还是右值来分派。

```cpp
void overload_ref(int &x);       // 1. 匹配左值（非常量）
void overload_ref(const int &x); // 2. 匹配常量左值
void overload_ref(int &&x);      // 3. 匹配右值（临时对象或 move 后的对象）
```

**匹配优先级：**
- `int a = 10; overload_ref(a);` -> 匹配 1。
- `const int b = 20; overload_ref(b);` -> 匹配 2。
- `overload_ref(30);` -> 优先匹配 3。如果没有 3，则会匹配 2（`const &` 可以绑定到右值）。

### 3.2 ⚠️ 重载的二义性 (Ambiguity)
当重载函数遇到默认参数时，极易产生歧义。
```cpp
void func(int a);
void func(int a, int b = 0);

// 调用处
func(10); // 编译报错！编译器不知道该调用哪一个。
```

## 4. 可变参数函数 (Variadic Functions)
C++ 提供两种方式处理数量不定的参数：传统的 C 风格和现代的 C++ 模板风格。

### 4.1 C 风格可变参数 (`...`)
- **头文件**：`<cstdarg>`
- **核心组件**：
  - `va_list`: 指向参数列表的指针。
  - `va_start(list, last_fixed_arg)`: 初始化，开始获取参数。
  - `va_arg(list, type)`: 按指定类型获取下一个参数，并移动指针。
  - `va_end(list)`: 结束清理。
- **局限性**：
  - **类型不安全**：编译器不检查参数类型，全靠程序员猜测。
  - **无法自动获知参数个数**：通常需要手动传入一个 `count` 或者通过格式化字符串（如 `printf` 中的 `%d`）解析。

### 4.2 现代 C++ 可变参数模板 (Variadic Templates)
- **语法**：使用 `template<typename... Args>` 定义。
- **优势**：
  - **类型安全**：编译时确定类型。
  - **灵活性**：支持不同类型的参数混合传递。
- **展开方式**：通常通过**递归**或 **折叠表达式 (Fold Expressions, C++17)** 展开参数包。
  - `sizeof...(args)`: 可以在编译时获取参数包中的参数个数。

---

## 5. 补充知识点：函数指针 vs 仿函数
- **函数指针**：`void (*p)(int)`，传统的调用方式。
- **仿函数 (Functor)**：重载了 `operator()` 的类对象，可以像函数一样被调用，且能保存状态（见 `1_yinyong_test` 案例）。

---

## 6. 总结建议
- **默认参数**尽量写在声明（头文件）中，方便调用者查阅。
- **函数重载**应保持逻辑一致性，不要让相同名字的函数做完全不相关的事情。
- **可变参数**：现代 C++ 项目中，应**优先使用可变参数模板**或 `std::initializer_list`，尽量避免使用 C 风格的省略号 `...`。
