# C++ 线程同步: Mutex 与 Locks 学习笔记

## 1. 互斥锁 (std::mutex)
`std::mutex` 用于保护共享数据，防止多个线程同时访问导致的数据竞争 (Data Race)。

### 常用操作
- `lock()`：加锁。
- `unlock()`：释放锁。
- `try_lock()`：尝试加锁，若已被占用则返回 false。

## 2. RAII 锁包装器
直接调用 `lock()` 和 `unlock()` 是危险的，因为如果程序在 `unlock()` 之前抛出异常或返回，锁将永远无法释放。C++ 提供了 RAII 风格的锁包装器。

### std::lock_guard
- **特点**：最简单的锁包装器。构造时加锁，析构时自动解锁。
- **使用场景**：不需要在作用域中手动解锁的场景。
- **限制**：不可复制、不可移动。

### std::unique_lock
- **特点**：比 `lock_guard` 更强大、更灵活。
- **功能**：
    - 支持手动调用 `lock()` 和 `unlock()`。
    - 支持 `std::defer_lock`（延迟加锁）。
    - 可以转移锁的所有权（可移动，不可复制）。
    - 是 `std::condition_variable` 必需的锁类型。
- **代价**：由于内部维护了锁的状态，其性能开销略大于 `lock_guard`。

## 3. 死锁 (Deadlock) 防范
### 什么是死锁？
当两个或多个线程互相等待对方释放锁时，就会发生死锁。

### 如何避免？
1. **加锁顺序一致**：始终按相同的顺序对多个互斥量加锁。
2. **std::lock**：C++11 提供了 `std::lock` 函数，可以同时锁定多个互斥量而不产生死锁。
   ```cpp
   std::lock(mtx1, mtx2);
   std::lock_guard<std::mutex> lk1(mtx1, std::adopt_lock);
   std::lock_guard<std::mutex> lk2(mtx2, std::adopt_lock);
   ```

## 4. 扩展补充
### std::recursive_mutex
允许同一个线程多次锁定同一个互斥量。常用于递归函数中。

### std::timed_mutex
支持带有超时限制的锁定尝试。
