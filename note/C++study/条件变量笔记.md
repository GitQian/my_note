# C++ 线程通信: 条件变量 (Condition Variable) 学习笔记

## 1. 什么是条件变量？
`std::condition_variable` 用于线程间的同步。它允许一个或多个线程等待某个条件满足，而无需不断地轮询（Polling）。

### 核心机制
- **等待 (wait)**：线程进入阻塞状态，释放它所持有的锁，直到被通知。
- **通知 (notify)**：另一个线程改变了条件，并发送信号唤醒等待中的线程。

## 2. 关键 API
- `wait(lock, predicate)`：阻塞当前线程直到被唤醒，且 `predicate` 返回 true。
- `notify_one()`：唤醒一个正在等待的线程。
- `notify_all()`：唤醒所有正在等待的线程。

## 3. 为什么必须配合 std::unique_lock？
`std::condition_variable` 的 `wait` 函数需要一个 `std::unique_lock<std::mutex>` 参数。
- 原因是 `wait` 在进入阻塞前需要**原子性地**释放互斥锁，并在被唤醒后重新获取该锁。只有 `unique_lock` 提供了这种灵活性。

## 4. 虚假唤醒 (Spurious Wakeup)
### 什么是虚假唤醒？
线程可能在没有被显式通知的情况下被唤醒，或者在通知后条件仍未满足时被唤醒。

### 解决方案
始终在循环中检查条件，或者使用 `wait` 的 Lambda 重载形式：
```cpp
cv.wait(lock, [] { return condition_met; });
// 相当于：
while (!condition_met) { cv.wait(lock); }
```

## 5. 生产者-消费者模式
这是条件变量最经典的应用场景：
- **生产者**：生产数据放入队列，并调用 `notify_one()`。
- **消费者**：在队列为空时调用 `wait()` 挂起，直到有数据被生产。

## 6. 补充：信号量 (Semaphore)
C++20 引入了 `std::counting_semaphore` 和 `std::binary_semaphore`。在 C++11/14/17 中，条件变量是实现信号量功能的标准方式。
