# C++ 异常处理 (Exception Handling) 学习笔记

异常处理是 C++ 处理运行时错误的重要机制，它通过分离错误的检测与处理逻辑，使程序更健壮。

---

## 1. 基础语法 (try, throw, catch)

- **`throw`**：检测到错误时，抛出一个异常对象。
- **`try`**：包含可能产生异常的代码块。
- **`catch`**：捕获特定类型的异常。

```cpp
try {
    throw std::runtime_error("Error!");
} catch (const std::runtime_error& e) {
    std::cerr << e.what() << std::endl;
}
```

---

## 2. 标准异常体系

C++ 提供了一套基于 `std::exception` 的派生类：
- `std::runtime_error`：运行时错误（如除 0、内存溢出）。
- `std::invalid_argument`：传入了无效参数。
- `std::out_of_range`：下标越界。
- `std::logic_error`：程序逻辑错误。

---

## 3. 自定义异常 (Custom Exception)

推荐继承 `std::exception` 并重写 `what()` 方法：
- **注意**：`what()` 方法应声明为 `const char* what() const noexcept override`。
- `noexcept` 保证了异常查询本身不会再抛出异常，这是安全稳定的关键。

---

## 4. RAII 与 栈回退 (Stack Unwinding)

**重要知识点**：当异常抛出时，程序会沿着函数调用链层层向上寻找匹配的 `catch`。
在这个过程中，**所有已经构造好的局部对象都会自动执行析构函数**。这就是 RAII（资源获取即初始化）在异常安全中的核心价值。

**示例：**
```cpp
class Resource {
    ~Resource() { /* 即使发生异常，这行代码也会执行，确保资源释放 */ }
};
```

---

## 5. 补充重要知识点

### 5.1 捕获所有类型 (`catch(...)`)
有时候我们无法预知会抛出什么类型（如代码中抛出了个 `int`），可以使用 `...` 捕获：
```cpp
catch (...) {
    // 处理所有未被上方 catch 捕获的异常
}
```

### 5.2 重新抛出异常 (`throw;`)
在 `catch` 块中处理完局部逻辑后，可以继续将异常抛给调用者：
```cpp
catch (const std::exception& e) {
    log(e);
    throw; // 重新抛出当前捕获的异常
}
```

### 5.3 异常安全保证 (Exception Safety Guarantees)
- **无异常保证**：函数承诺绝不抛出异常。
- **强异常保证**：如果函数抛出异常，程序状态会回滚到调用前的状态（“全有或全无”）。
- **基本异常保证**：抛出异常后，程序仍处于合法状态，没有资源泄漏，但数据可能已部分修改。

### 5.4 性能开销
异常抛出和捕获会有性能开销（尤其是查找匹配的 `catch` 块）。因此，**异常应仅用于处理非预期的、真正的“异常”情况**，不应作为常规的控制流（如替代 `if-else`）。
